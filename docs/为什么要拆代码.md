为什么要将sketch.js拆到不同的代码中？
1.降低耦合度（代码之间的依赖程度）。

之前所有的功能都写在一个文件中，导致在多人协作开发会很麻烦。举个例子：同学A在第200行新写了50行代码，在第500行删除了30行代码，然后提交到github上。此时同学B再打开文件，发现①代码的相对位置变了，需要花一些时间重新找自己负责的那块代码；②不确定同学A的增改是否会影响到他负责的那部分模块，需要阅读同学A的改动日志，增加了调试难度。

因此，将一个大文件拆分成若干中小文件，每人只负责其中几个文件，可以降低工程复杂度。

2.将代码封装成一个接口，降低开发难度。负责每个模块的人只需要知道如何调用封装好的接口就行，不需要阅读大段代码来了解是如何这部分生效的。



拆分后的文件结构：

asset/ 统一存放音效、背景、贴图等资源
js/

​	sketch.js 项目入口，只保留必要函数，用于调用其他js文件的函数

​	core/

​		config.js 全局常量

​		state.js 游戏运行时状态

​		asset-loader.js 加载asset下的资源

​	data/ 定义配置表，可以看作这是存放每张表表头的地方。如enemy-pool.js类定义了敌人的类型、形状、速度、血量等属性

​	entity/ 定义对象。如enemy.js中定义了创建敌人，伤害敌人和敌人朝玩家移动三个函数

​	scene/ 场景的切换，开始界面，游戏界面，暂停界面等

​	system/ 系统的整体逻辑，从sketch.js拆分下的代码大部分都在这里

​		camera 摄像机

​		combat 敌人、子弹与玩家的碰撞处理

​		input 键盘和鼠标输入

​		progression 当前游戏进度

​		spawn 敌人生成

​		ui 绘制
